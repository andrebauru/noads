# 📋 Noads Contributor Guide

## Bem-vindo! 👋

Obrigado por se interessar em contribuir com Noads! Este documento fornece diretrizes para contribuições.

## 🎯 Como Contribuir

### Reportar Bugs
1. Verifique se o bug já foi reportado
2. Abra uma [Issue](https://github.com/seu-usuario/Noads/issues)
3. Descreva:
   - Comportamento esperado vs atual
   - Passos para reproduzir
   - Seu ambiente (OS, Browser, Python version)

### Sugerir Features
1. Abra uma [Discussion](https://github.com/seu-usuario/Noads/discussions)
2. Descreva a feature e por que seria útil
3. Discuta com a comunidade

### Fazer Pull Request
1. Fork o repositório
2. Crie uma branch (`git checkout -b feature/sua-feature`)
3. Commit suas mudanças (`git commit -m 'Descrição clara'`)
4. Push para a branch (`git push origin feature/sua-feature`)
5. Abra um Pull Request

## 📝 Diretrizes de Código

- Use nomes descritivos para variáveis/funções
- Adicione comentários em código complexo
- Mantenha funções pequenas e focadas
- Teste suas mudanças

## 🧪 Antes de Fazer PR

```bash
# Teste localmente
python simple_api.py
php -S 127.0.0.1:8000

# Verifique no browser
# DevTools Console para erros
```

## 💬 Comunidade

- Issues para bugs/features
- Discussions para dúvidas
- Respeito e gentileza sempre
